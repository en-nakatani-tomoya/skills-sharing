# ADR-004: フラット構造の採用とスキル所有権モデル

| 項目 | 値 |
|---|---|
| ステータス | 採用 |
| 作成日 | 2026-03-19 |
| 関連 ADR | 上書き: [ADR-002](./002-monorepo-design.md) のリポジトリ構造・利用方法の節を改定 |

---

## コンテキスト

ADR-002 の Monorepo 設計では、ユーザーディレクトリ配下を `claude/skills/`、`cursor/skills/` 等エージェント別に分ける構造だった。運用を検討する中で、以下の問題が明らかになった:

1. スキルはエージェント間で共通（和集合）であり、エージェント別にディレクトリを分ける必然性がない
2. 階層が深く、GitHub Web UI でのブラウズ体験が悪い
3. 他人のスキルを利用する際の所有権モデルが未定義だった

## 決定

### 1. ディレクトリ構造をフラットにする

エージェント別の階層を廃止し、ユーザーディレクトリ直下にスキルをフラットに配置する。

**変更前（ADR-002）:**

```
users/sample-user/
├── claude/
│   └── skills/
│       ├── dev-workflow/SKILL.md
│       └── pr-review/SKILL.md
└── cursor/
    └── skills/
        └── refactor-guide/SKILL.md
```

**変更後:**

```
users/sample-user/
├── README.md
├── dev-workflow/SKILL.md
├── pr-review/SKILL.md
└── refactor-guide/SKILL.md
```

### 2. 自分のスキルはシンボリックリンクで全エージェントに配置する

```bash
ln -s ~/skills-sharing/users/<your-name> ~/.claude/skills
ln -s ~/skills-sharing/users/<your-name> ~/.cursor/skills
```

シンボリックリンク 1 本で全スキルが全エージェントで使える。スキルの追加・削除時にリンクの張り直しが不要。

### 3. 他人のスキルはコピー（フォーク）で取り込む

他人のスキルを使いたい場合、シンボリックリンクではなく自分のディレクトリにコピーして取り込む。これは Git のフォークモデルと同じ発想。

| 観点 | シンボリックリンク（参照） | コピー（フォーク） |
|---|---|---|
| カスタマイズ | 元のスキルに変更が反映されてしまう | 自由にカスタマイズできる |
| 所有権 | 曖昧 | 明確（自分のディレクトリにあるものは自分のもの） |
| 元スキルの更新追従 | 自動（意図しない変更を受ける可能性） | 意図的に再コピーする必要がある |
| 壊れるリスク | 元の所有者が削除・改名するとリンク切れ | 独立しているので影響なし |

コピーでの取り込みにより、「何を Skill に行わせたいかが利用者によって変わる」問題（[ヘンリー社の事例](../references/henry-skill-management.md)で指摘）を回避できる。改善が生まれた場合は、PR ではなく知見の共有（Slack、勉強会）として還元する。

## 影響・トレードオフ

- `skills-browse` の install コマンドは「エージェントのローカルに配置する」ではなく「他人のスキルを自分のディレクトリにフォークする」操作になる
- starter-kit のスキルも自分のディレクトリにコピーして使う形になる（starter-kit は出発点の見本であり、直接参照するものではない）
- 元スキルの更新に自動追従しないため、改善を取り込むには意図的な再コピーが必要。これはトレードオフとして許容する
